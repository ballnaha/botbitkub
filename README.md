# Bitkub Mini Bot - คู่มือการใช้งานและการตั้งค่า `.env`

บอทเทรด Bitkub อัตโนมัติ (Bitkub Mini Bot) มาพร้อมกับระบบสแกนสัญญาณเทคนิคัล และระบบกรองสัญญาณด้วย AI Review (ปัจจุบันรองรับ Gemini)

---

## 📌 วิธีการตั้งค่าไฟล์ `.env` (สำคัญที่สุด)

ในการใช้งานบอทครั้งแรก คุณจำเป็นต้องสร้างและแก้ไขไฟล์ชื่อ `.env` เพื่อใช้สำหรับเก็บรหัสลับและการตั้งค่าส่วนตัว โดยไฟล์นี้จะถูกบันทึกไว้ใน**โฟลเดอร์เดียวกันกับตัวโปรแกรม** `BitkubMiniBot.exe`

### 1. วิธีสร้างไฟล์ `.env`
หากคุณยังไม่มีไฟล์ `.env` ในโฟลเดอร์ `BitkubMiniBot` ให้ทำตามขั้นตอนดังนี้:
1. คลิกขวาในพื้นที่ว่างของโฟลเดอร์ `BitkubMiniBot` เลือก **New (สร้างใหม่)** -> **Text Document (เอกสารข้อความ)**
2. ตั้งชื่อไฟล์เป็น `.env` (ตรวจสอบว่าไม่มีนามสกุล `.txt` ต่อท้าย หากระบบซ่อนนามสกุลไฟล์อยู่ ให้ตั้งเป็น `.env`)
3. เปิดไฟล์ด้วยโปรแกรม Notepad หรือ Text Editor อื่น ๆ
4. คัดลอกข้อความตั้งค่าจากตัวอย่างด้านล่างนี้ไปใส่ จากนั้นทำการแก้ไขค่าให้ตรงกับบัญชีของคุณ

---

### 2. ตัวอย่างข้อมูลในไฟล์ `.env`
คัดลอกส่วนนี้ไปใส่ในไฟล์ `.env` ของคุณ:

```env
# API Key และ API Secret จากบัญชี Bitkub ของคุณ
BITKUB_API_KEY=ใส่_API_KEY_ที่นี่
BITKUB_API_SECRET=ใส่_API_SECRET_ที่นี่

# ชื่อเจ้าของบัญชีสำหรับแสดงผลบนหน้าจอแดชบอร์ด
BITKUB_OWNER_NAME=ชื่อของคุณ

# ยืนยันตัวตนสำหรับเข้าหน้าเว็บ Dashboard (HTTP Basic Auth)
DASHBOARD_USERNAME=admin
DASHBOARD_PASSWORD=กำหนดรหัสผ่านของคุณเองที่นี่

# API Key ของผู้ให้บริการ AI ที่ระบบรองรับปัจจุบัน (Gemini,Deepseek) สำหรับให้ AI ช่วยกรองสัญญาณซื้อขาย
GEMINI_API_KEY=ใส่_GEMINI_API_KEY_ที่นี่
DEEPSEEK_API_KEY=ใส่_DEEPSEEK_API_KEY_ที่นี่่
# การตั้งค่าพอร์ตและโฮสต์ (ปกติใช้ค่าเริ่มต้นได้เลย)
HOST=127.0.0.1
PORT=8282
```

---

### 3. อธิบายรายละเอียดตัวแปรใน `.env`

| ชื่อตัวแปร | ความหมาย / รายละเอียด |
| :--- | :--- |
| `BITKUB_API_KEY` | **API Key** ที่ได้จากเว็บ Bitkub (ต้องเลือกสิทธิ์ Read และ Write เพื่อส่งคำสั่งซื้อขาย) |
| `BITKUB_API_SECRET` | **API Secret** ที่คู่กับ API Key ด้านบน (ระวัง! ห้ามเผยแพร่ให้ใครรู้เด็ดขาด) |
| `BITKUB_OWNER_NAME` | ชื่อของคุณ เพื่อแสดงความยินดีต้อนรับที่มุมบนของ Dashboard (เช่น `BITKUB_OWNER_NAME=Thanya`) |
| `DASHBOARD_USERNAME` | ชื่อผู้ใช้สำหรับ Login เข้าหน้า Dashboard (ค่าเริ่มต้นคือ `admin`) |
| `DASHBOARD_PASSWORD` | รหัสผ่านสำหรับ Login (แนะนำให้เปลี่ยนเพื่อความปลอดภัย โดยเฉพาะเวลาเอาไปรันบน VPS) |
| `GEMINI_API_KEY` | **API Key ของ Gemini** เพื่อใช้ฟีเจอร์ AI Review กรองสัญญาณซื้อขาย (ถ้าจะใช้ผู้ให้บริการ AI อื่น ต้องเพิ่ม adapter ในโค้ดก่อน) |
| `HOST` | IP Address สำหรับรัน Backend (ค่าเริ่มต้น `127.0.0.1` สำหรับรันบนเครื่องตัวเอง) |
| `PORT` | Port ที่ Backend ใช้รัน (ค่าเริ่มต้นคือ `8282`) |

## 🚀 วิธีการใช้งานโปรแกรม

1. ตรวจสอบให้แน่ใจว่าไฟล์ `.env` อยู่ในโฟลเดอร์เดียวกับ `BitkubMiniBot.exe`
2. ดับเบิ้ลคลิกไฟล์ `BitkubMiniBot.exe` เพื่อเริ่มการใช้งาน
3. ระบบจะเปิดหน้าต่างโปรแกรมขึ้นมา และคุณสามารถ Login เข้าหน้าระบบด้วย **Username** และ **Password** ที่ตั้งไว้ในไฟล์ `.env`
4. หน้าเว็บแดชบอร์ดสามารถเปิดผ่านบราวเซอร์ปกติได้เช่นกันที่ URL: `http://127.0.0.1:8282`

## ⚠️ ข้อควรระวังด้านความปลอดภัย
* **ห้ามแชร์ไฟล์ `.env` ให้กับผู้อื่นเด็ดขาด!** เพราะในนั้นมี API Key และ Secret ของ Bitkub ซึ่งหากผู้อื่นได้ไป จะสามารถเข้าถึงบัญชีและสั่งซื้อขายแทนได้
* หากย้ายโปรแกรมไปรันบนเซิร์ฟเวอร์จำลอง (VPS) หรือเครื่องอื่น อย่าลืมย้ายไฟล์ `.env` ตามไปด้วย

## 🧠 กลยุทธ์การเทรด (Trading Strategy)

### สถาปัตยกรรมการตัดสินใจ

```
แท่งเทียน 15 นาที (OHLCV)
        ↓
กลยุทธ์เทคนิคัล → สัญญาณซื้อ/ขาย
        ↓
AI Gate (Gemini / DeepSeek) — ถ้าเปิดใช้
        ↓
ส่งคำสั่งซื้อ / ขาย → Bitkub V3 API
```

บอททำงานด้วย **2 loop แยกเธรด** เพื่อให้ทั้งแม่นและไว:

| Loop | รอบ | หน้าที่ |
|------|-----|--------|
| **Scan loop** | ทุก 60 วิ | สแกน 18 คู่เหรียญบนแท่งเทียน 15 นาที — หาสัญญาณ **ซื้อ/ขายตามกลยุทธ์** |
| **Price loop** | ทุก ~5 วิ | ดึงราคา real-time (เฉพาะตอนมี position) — อัปเดต PnL + เช็ค **SL / Trailing / TP** แบบไว |

> Price loop ดึง ticker ครั้งเดียวได้ราคาทุกคู่ (cost คงที่) และเขียน DB เฉพาะตอนค่าจำเป็นเปลี่ยน
> ฝั่งเว็บอ่าน PnL จาก memory ที่ price loop รีเฟรชไว้ — **ไม่ยิง Bitkub เองใน request path จึงไม่กระตุก**
> (ฐานข้อมูลเปิด SQLite WAL mode ให้ web อ่านได้ลื่นแม้ background กำลังเขียน)

### กลยุทธ์ที่มีให้เลือก 3 แบบ

#### ⚖️ 1. Multi-Indicator — Balanced Signal (ค่าเริ่มต้น)

**ความเสี่ยง:** ปานกลาง | **อินดิเคเตอร์:** RSI(14), Bollinger Bands(20, 2σ), EMA 9/21, Volume SMA(20)

**ซื้อเมื่อ:** ผ่าน **≥ 2 ใน 4** เงื่อนไข (Scoring System)
1. RSI < 40 — oversold
2. ราคา Close < Bollinger Band ล่าง
3. EMA9 > EMA21 — เทรนด์ขาขึ้นระยะสั้น
4. Volume > Volume SMA20

**ขายเมื่อ:** ผ่าน **ข้อใดข้อหนึ่ง**
1. RSI > 70
2. ราคา Close > Bollinger Band บน
3. EMA9 < EMA21

> หลักการ: เข้ายาก ออกง่าย — ใช้ confirmation หลายตัวกรองสัญญาณหลอก

---

#### 🛡️ 2. MACD + RSI — Safe Reversal

**ความเสี่ยง:** ต่ำ | **อินดิเคเตอร์:** MACD(12, 26, 9), RSI(14)

**ซื้อเมื่อ:** ต้องผ่าน **ทั้ง 3 เงื่อนไขพร้อมกัน**
1. MACD Line ตัดขึ้นเหนือ Signal Line (Golden Cross บนแท่งนี้)
2. MACD Line < 0 — ยังอยู่ในโซนลบ (early reversal)
3. RSI < 55

**ขายเมื่อ:** ผ่าน **ข้อใดข้อหนึ่ง**
1. MACD Line ตัดลงใต้ Signal Line (Death Cross)
2. RSI > 70

> หลักการ: เทรดน้อย แต่แม่น — รอจังหวะที่ครบ 3 เงื่อนไขจึงเข้า

---

#### 🚀 3. Aggressive Momentum — Fast Breakout

**ความเสี่ยง:** สูง | **อินดิเคเตอร์:** EMA 5/13, RSI(14), 12-Candle Breakout, Volume SMA(12)

**ซื้อเมื่อ:** ต้องผ่าน **ทั้ง 5 เงื่อนไขพร้อมกัน**
1. ราคา Close ทะลุเหนือ High สูงสุด 12 แท่งก่อนหน้า (Breakout)
2. แท่งก่อนหน้ายังไม่ได้ Breakout (fresh breakout เท่านั้น)
3. EMA5 > EMA13 — เทรนด์ขาขึ้น
4. RSI อยู่ในช่วง 48–78 (momentum zone)
5. Volume > 1.15× Volume SMA — แรงซื้อหนุน

**ขายเมื่อ:** ผ่าน **ข้อใดข้อหนึ่ง**
1. RSI > 82 — momentum หมด
2. Close < EMA5 — หลุดเทรนด์สั้น
3. Close < Low ต่ำสุด 6 แท่งก่อนหน้า — pullback รุนแรง

> หลักการ: ตามโมเมนตัม ซื้อเมื่อราคาทะลุจุดต้านระยะสั้น เทรดบ่อย กำไรไว ผันผวนสูง

### การจัดการความเสี่ยง (Risk Management)

| พารามิเตอร์ | ค่าเริ่มต้น | รายละเอียด |
|-------------|------------|-----------|
| Stop Loss | -5% | หยุดขาดทุนอัตโนมัติ |
| Take Profit | +10% | ทำกำไรอัตโนมัติ (RR 1:2) |
| Trailing Stop | เปิด (+3% / -2%) | เริ่ม trail เมื่อกำไรถึง +3% แล้วขายเมื่อราคาย่อจากจุดสูงสุด 2% — ล็อกกำไรกันกรณีวิ่งขึ้นแล้วหลุดกลับไปโดน SL |
| Cooldown | เปิด (30 นาที) | หลังขายขาดทุน พักการเข้าซื้อเหรียญเดิม 30 นาที กัน whipsaw ในตลาด sideways |
| Max Open Trades | 3 | จำนวน position สูงสุดพร้อมกัน |
| Stake per Trade | 100 THB | เงินต่อไม้ |
| Max Budget | 5,000 THB | เพดานงบรวมทุก position |
| ค่าธรรมเนียม | 0.25% | หักทั้งซื้อและขาย (Bitkub standard) |

**ลำดับตรวจสอบขาย:** สัญญาณกลยุทธ์ → Stop Loss → Trailing Stop → Take Profit

#### ⚙️ การตั้งค่า Trailing Stop & Cooldown (ใน `bot_config.json`)

| คีย์ | ค่าเริ่มต้น | ความหมาย |
|------|------------|----------|
| `trailing_stop_enabled` | `true` | เปิด/ปิด Trailing Stop |
| `trailing_activation_pct` | `3.0` | เริ่ม trail เมื่อกำไรถึงกี่ % |
| `trailing_stop_pct` | `2.0` | ขายเมื่อราคาย่อจากจุดสูงสุดกี่ % |
| `cooldown_enabled` | `true` | เปิด/ปิด Cooldown |
| `cooldown_minutes` | `30` | ระยะเวลาพัก (นาที) |
| `cooldown_after_loss_only` | `true` | นับ cooldown เฉพาะตอนขาดทุน (ถ้า `false` จะพักหลังขายทุกครั้ง) |

#### 🐻 Market Regime Filter — เกราะตลาดหมี (ครอบทุกกลยุทธ์)

ก่อนเข้าซื้อทุกครั้ง บอทจะเช็ค **สภาพตลาดรวม** จากเหรียญอ้างอิง (BTC) เทียบกับ EMA ระยะยาวบน TF ใหญ่ — ถ้าตลาดเป็นหมี (BTC ต่ำกว่า EMA) จะ **งดเข้าซื้อ** หรือ **ลด stake** ให้ทุกกลยุทธ์โดยอัตโนมัติ ป้องกันบอทไล่ซื้อตอนตลาดพังทั้งกระดาน

> 💡 **ยกเว้นกลยุทธ์ขาลง** (market_condition = `downtrend` เช่น Bounce Scalper) — ฟิลเตอร์จะไม่บล็อก เพราะกลยุทธ์เหล่านี้ออกแบบมาเล่นตลาดหมีโดยตรง
> ผล regime ถูก cache 15 นาที และ **fail-open** (ไม่บล็อก) เมื่อดึงข้อมูล BTC ไม่ได้ เพื่อไม่ให้บอทค้าง

| คีย์ | ค่าเริ่มต้น | ความหมาย |
|------|------------|----------|
| `regime_filter_enabled` | `true` | เปิด/ปิดเกราะตลาดหมี |
| `regime_reference_symbol` | `BTC/THB` | เหรียญอ้างอิงสภาพตลาดรวม |
| `regime_timeframe` | `240` | TF ที่ใช้ประเมิน (นาที, 240 = 4 ชม.) |
| `regime_ema_period` | `200` | คาบ EMA สำหรับเส้นแบ่ง bull/bear |
| `regime_action` | `block` | `block` = งดซื้อ, `reduce` = ลด stake |
| `regime_reduce_factor` | `0.5` | สัดส่วน stake เมื่อใช้โหมด `reduce` |

### AI Gate — ชั้นกรองสัญญาณเสริม

เปิด/ปิดได้ด้วยการตั้งค่า `ai_enabled` — AI **ไม่ได้สร้างสัญญาณเอง** แต่ทำหน้าที่ตรวจสอบหลังกลยุทธ์ให้สัญญาณซื้อแล้ว

**ผู้ให้บริการ:** Gemini (default: `gemini-3.5-flash`), DeepSeek (`deepseek-reasoner`)

**เกณฑ์อนุมัติ (ต้องผ่านครบทั้ง 3):**
- `decision` = `buy`
- `score` ≥ 65 (0–100)
- `confidence` ≥ 0.55 (0.0–1.0)

AI จะตอบกลับเหตุผลเป็น**ภาษาไทย** เสมอ และมี rate limit ในตัว (12 req/นาที)
